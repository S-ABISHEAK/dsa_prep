# 141. Linked List Cycle

## Problem

Given the head of a linked list, determine if the linked list contains a cycle.

### Constraints

```text
0 <= number of nodes <= 10^4
-10^5 <= Node.val <= 10^5
pos is -1 or a valid index
```

### Example

Input:

```text
head = [3,2,0,-4]
pos = 1
```

Output:

```text
true
```

---

## Code

```java
public class Solution {
    public boolean hasCycle(ListNode head) {
        ListNode fast = head; 
        ListNode slow = head;

        while(fast != null && fast.next != null){
            slow = slow.next;
            fast = fast.next.next;

            if(fast == slow){
                return true;
            }
        }

        return false;
    }
}
```

---

## Code Explanation

### Initialize Two Pointers

```java
ListNode fast = head;
ListNode slow = head;
```

Visual:

```text
head
 ↓
[3] → [2] → [0] → [-4]
 ↑               ↓
 └───────────────┘

slow = head
fast = head
```

- `slow` moves 1 step at a time.
- `fast` moves 2 steps at a time.

---

### Move the Pointers

```java
slow = slow.next;
fast = fast.next.next;
```

Visual:

```text
Step 1

slow
 ↓
[3] → [2] → [0] → [-4]
          ↑
         fast
```

```text
Step 2

          slow
            ↓
[3] → [2] → [0] → [-4]
      ↑
     fast
```

- Slow moves by 1 node.
- Fast moves by 2 nodes.

---

### Check if They Meet

```java
if(fast == slow){
    return true;
}
```

Visual:

```text
[3] → [2] → [0] → [-4]
          ↑
     slow, fast
```

- If a cycle exists, fast will eventually catch slow.
- Meeting point confirms a cycle.

---

### No Cycle Found

```java
while(fast != null && fast.next != null)
```

Visual:

```text
[1] → [2] → [3] → null
                     ↑
                    fast
```

- If `fast` reaches `null`, the list ends.
- Therefore, no cycle exists.

---

## Why Does This Work?

```text
Slow moves 1 step.
Fast moves 2 steps.

In a cycle, fast gains 1 node on slow every move.

Eventually, fast catches slow.
```

Think of two runners on a circular track:

```text
Slow Runner  -> 1 step
Fast Runner  -> 2 steps

Fast will eventually lap Slow.
```

---

## Time Complexity

```text
O(n)
```

- Each node is visited at most a few times.

---

## Space Complexity

```text
O(1)
```

- Only two pointers are used.

---

## Pattern

Floyd's Cycle Detection (Tortoise and Hare)

---

## When To Use This Pattern

- Detect cycles in Linked Lists.
- Find the starting node of a cycle (Linked List Cycle II).
- Problems involving repeated traversal in a looped structure.

```

### Quick Revision

```text
slow = slow.next
fast = fast.next.next

If slow == fast → Cycle exists
If fast reaches null → No cycle
```

### Core Idea

```text
Slow moves 1 step
Fast moves 2 steps

Cycle Present  → They meet
No Cycle       → Fast reaches null
```

This is one of the most important Linked List interview patterns and is known as **Floyd's Tortoise and Hare Algorithm**.
