# 142. Linked List Cycle II

## Problem

Given the head of a linked list, return the node where the cycle begins. If there is no cycle, return `null`.

### Constraints

```text
0 <= number of nodes <= 10^4
-10^5 <= Node.val <= 10^5
pos is -1 or a valid index
```

### Example

Input:

```text
head = [3,2,0,-4]
pos = 1
```

Output:

```text
Node with value 2
```

---

## Code

```java
public class Solution {
    public ListNode detectCycle(ListNode head) {
        if(head == null || head.next == null) return null;

        ListNode slow = head;
        ListNode fast = head;

        while(fast != null && fast.next != null){
            slow = slow.next;
            fast = fast.next.next;

            if(slow == fast){
                ListNode temp = head;

                while(temp != slow){
                    slow = slow.next;
                    temp = temp.next;
                }

                return slow;
            }
        }

        return null;
    }
}
```

---

## Code Explanation

### Step 1: Detect Cycle

```java
slow = slow.next;
fast = fast.next.next;
```

Visual:

```text
head
 ↓
[3] → [2] → [0] → [-4]
       ↑         ↓
       └─────────┘

slow -> 1 step
fast -> 2 steps
```

* If a cycle exists, `slow` and `fast` will eventually meet.
* This is Floyd's Cycle Detection Algorithm.

---

### Step 2: Meeting Point Found

```java
if(slow == fast)
```

Visual:

```text
[3] → [2] → [0] → [-4]
       ↑     ↓
       └─────┘

      slow
       ↓
      fast
```

* Meeting point is inside the cycle.
* It is NOT necessarily the cycle start.

---

### Step 3: Find Cycle Start

```java
ListNode temp = head;
```

Visual:

```text
temp
 ↓
[3] → [2] → [0] → [-4]
       ↑         ↓
       └─────────┘
          ↑
        slow
```

* Put one pointer at `head`.
* Keep the other pointer at the meeting point.

---

### Step 4: Move Both One Step

```java
while(temp != slow){
    temp = temp.next;
    slow = slow.next;
}
```

Visual:

```text
Step 1

temp
 ↓
[3] → [2] → [0] → [-4]
       ↑         ↓
       └─────────┘
              ↑
            slow


Step 2

       temp
         ↓
[3] → [2] → [0] → [-4]
       ↑         ↓
       └─────────┘
         ↑
       slow
```

* Move both pointers one step at a time.
* They meet exactly at the cycle starting node.

---

### Step 5: Return Answer

```java
return slow;
```

Visual:

```text
[3] → [2] → [0] → [-4]
       ↑         ↓
       └─────────┘
       ↑
   Cycle Start
```

* The meeting point of `temp` and `slow` is the start of the cycle.

---

## Why Does This Work?

```text
Let:

L = distance from head to cycle start
C = distance from cycle start to meeting point

When slow and fast meet:

2 × (L + C) = L + C + n × cycleLength

⇒ L = n × cycleLength - C
```

This means:

```text
Distance from Head to Cycle Start
=
Distance from Meeting Point to Cycle Start
```

Therefore:

* One pointer starts at head.
* One pointer starts at meeting point.
* Move both one step.
* They meet at cycle start.

---

## Time Complexity

```text
O(n)
```

* One traversal to detect cycle.
* One traversal to find cycle start.

---

## Space Complexity

```text
O(1)
```

* Only pointers are used.

---

## Pattern

Floyd's Cycle Detection + Cycle Entry Point

---

## When To Use This Pattern

* Find the starting node of a cycle.
* Detect loops in linked structures.
* Problems involving repeated traversal and cycle detection.
* Follow-up of Linked List Cycle (LeetCode 141).
