# 206. Reverse Linked List

## Problem

Given the head of a singly linked list, reverse the list and return the new head.

### Constraints

```text id="k3iv08"
0 <= number of nodes <= 5000
-5000 <= Node.val <= 5000
```

### Example

Input:

```text id="ztxybd"
head = [1,2,3,4,5]
```

Output:

```text id="3qz6l0"
[5,4,3,2,1]
```

---

## Code

```java id="vhcvag"
class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode curr = head;

        while(curr != null){
            ListNode temp = curr.next;
            curr.next = prev;
            prev = curr;
            curr = temp;
        }

        return prev;
    }
}
```

---

## Code Explanation

### Initialize Pointers

```java id="8p7w4v"
ListNode prev = null;
ListNode curr = head;
```

Visual:

```text id="7m6c34"
prev = null

curr
 ↓
[1] → [2] → [3] → [4] → [5] → null
```

* `prev` stores the previous node.
* `curr` is the current node being processed.

---

### Store Next Node

```java id="e4qifv"
ListNode temp = curr.next;
```

Visual:

```text id="yr8jzq"
prev   curr   temp
 ↓      ↓      ↓
null   [1] → [2] → [3]
```

* Save the next node before changing links.
* Otherwise the remaining list will be lost.

---

### Reverse the Link

```java id="q1c3m0"
curr.next = prev;
```

Before:

```text id="8l0q4f"
null    [1] → [2] → [3]
        curr
```

After:

```text id="2xq6o5"
null ← [1]    [2] → [3]
        curr
```

* Reverse the direction of the pointer.

---

### Move Pointers Forward

```java id="swmyxv"
prev = curr;
curr = temp;
```

Visual:

```text id="zg2a5m"
After 1st Iteration

prev
 ↓
[1] → null

curr
 ↓
[2] → [3] → [4] → [5]
```

* `prev` moves to current node.
* `curr` moves to next node.

---

### Continue Until End

```text id="83j18x"
Iteration 1:
1 ← null

Iteration 2:
2 ← 1 ← null

Iteration 3:
3 ← 2 ← 1 ← null

Iteration 4:
4 ← 3 ← 2 ← 1 ← null

Iteration 5:
5 ← 4 ← 3 ← 2 ← 1 ← null
```

---

### Return New Head

```java id="h5eovl"
return prev;
```

Visual:

```text id="m9sgys"
prev
 ↓
[5] → [4] → [3] → [2] → [1] → null
```

* After the loop ends, `prev` points to the new head.

---

## Time Complexity

```text id="4p4owc"
O(n)
```

* Visit every node exactly once.

---

## Space Complexity

```text id="fngb1f"
O(1)
```

* Only pointers are used.

---

## Pattern

Linked List Pointer Manipulation

---

## When To Use This Pattern

* Reversing a linked list.
* Reversing a part of a linked list.
* Problems involving pointer direction changes.
* Foundation for Reverse K Groups, Palindrome Linked List, and Reverse Between.
